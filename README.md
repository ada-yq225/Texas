# 单机德州扑克

一个可本地运行的单机 Texas Hold'em 德州扑克练习游戏，支持 4-8 人桌、正常模式、教学全明牌模式、实时胜率、3Bet、All-in、All-in 发两次、全场盈亏统计和多风格机器人。

## 快速开始

```bash
python3 -m http.server 5173
```

然后打开：

```text
http://localhost:5173/
```

项目是纯前端静态应用，不需要安装 npm 依赖。

## 文件结构

```text
index.html              页面结构
styles.css              牌桌、座位、筹码和响应式 UI
game.js                 游戏规则、发牌、下注、机器人逻辑、胜率计算
vendor/pokersolver.js   手牌牌型比较库
```

## 主要功能

- 4-8 人桌可切换。
- 初始筹码可选：1000、2000、5000、10000。
- 盲注可选：5/10、10/20、25/50。
- 正常模式：只显示玩家手牌，机器人手牌隐藏。
- 教学模式：全明牌显示，并在每个玩家手牌上显示当前胜率。
- 实时胜率：根据当前公共牌和剩余牌堆进行蒙特卡洛模拟。
- 下注动作：弃牌、看牌/跟注、下注/加注、3Bet、半池下注、All-in。
- All-in 发牌：支持发一次或发两次。
- 全场盈亏：根据玩家当前筹码和初始买入统计。

## 机器人逻辑

机器人不是固定公式行动，而是由多组因素共同决策：

- 手牌胜率：`estimateEquity()` 会按当前街道估算机器人真实胜率。
- 机器人风格：在 `BOT_BLUEPRINTS` 中配置，包括紧度、激进度、诈唬率、慢打倾向、跟注倾向、位置意识和波动性。
- 玩家画像：`heroProfile()` 会根据玩家历史行为估算玩家激进度、跟注频率、面对下注弃牌率等。
- 牌桌环境：会考虑当前底池、跟注成本、SPR、位置、多方底池、公共牌湿度。
- 随机扰动：每个机器人每手牌会有 mood 波动，让行动不会完全重复。
- 加注标签：根据本轮加注次数显示 3Bet、4Bet、5Bet、All-in 等压力动作。

核心入口在：

```text
game.js -> decideForBot(player)
```

如果后续要接 AI API，可以把 `decideForBot(player)` 改成异步决策层，把当前牌局状态序列化后发送给模型，再让模型返回：

```json
{
  "type": "fold | check | call | raise | allin",
  "target": 120,
  "reason": "范围优势、位置优势、玩家最近过度弃牌"
}
```

建议保留本地规则引擎作为兜底，避免 API 超时或返回非法动作时游戏卡死。

## 胜率算法

教学模式下显示的是“当前最准确可用胜率”：

- 翻前、翻牌、转牌：从剩余牌堆中随机补全未知公共牌和未知对手手牌，进行多轮蒙特卡洛模拟。
- 河牌：牌面已经完整，直接比较所有未弃牌玩家牌型。
- 手牌比较由 `vendor/pokersolver.js` 完成。
- 胜率缓存由 `equityKey()` 控制，同一牌面不会重复计算。

相关代码：

```text
game.js -> calculateStudyEquities()
game.js -> simulateEquity()
game.js -> equityKey()
```

## UI 适配

桌面端使用一屏式布局：

```text
顶部数据栏
牌局设置栏
自适应牌桌
玩家手牌区
操作区
```

牌桌会根据浏览器宽高自动压缩座位、手牌、公共牌和操作区，避免出现玩家手牌看不到或座位重叠的问题。

## 后续商业化建议

- 增加开局动画、赢池动画和筹码飞行动画。
- 增加局后复盘：记录每手牌的胜率变化和 EV 决策点。
- 增加 AI 教练模式：每次玩家行动后解释更优选项。
- 增加难度档位：娱乐局、常规局、高手局。
- 增加本地存档：保存 bankroll、手数、胜率、VPIP、PFR、3Bet 等统计。
